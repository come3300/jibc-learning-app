    コマンドラインの動作確認
    それでは、コマンドライン端末が問題なく準備できたかどうかを確認します。
    コマンドライン端末を起動すると、文字入力ができるので、以下のように入力してみてください。

    ```
    bash-3.2$ python3 --version
    ```

    ここでひとつ注意があります。先頭のbash-3.2$の部分は入力しないでください。
    これは「端末が文字入力を求めていること」を示す記号で、「プロンプト」と呼びます。
    皆さんの端末にも、文字入力が可能なところの前に何かしら表示されていると思います（表示は端末によって異なります）。
    このテキストでは、「プロンプトの後に入力してほしい」という意図を明確にするためにプロンプトを含めて記載していますが、
    実際に入力するものではないことを覚えておいてください。
    また、入力は必ず半角英数字の入力モードで入力してください。全角文字ではうまく動きません。
    また、スペースは重要です。pythonと--versionの間には必ずスペースを入力してください。
    入力したら、最後にエンターキーを押してください。
    何が表示されたでしょうか。以下のような表示がされれば、
    コマンドライン端末の準備はバッチリです。

    ```
    Python 3.8.5
    ```
    数字の部分などは微妙に違うかもしれませんが、大体 ↑ のような表示になっていれば大丈夫です。
    もしこれとぜんぜん違う表示になっていたら質問してください。
    これは Python という言語の処理系のバージョンを示した文字が表示されています。
    ↑ の例では、3.8.5 というバージョンがインストールされていることがわかります。
    これでコマンドライン端末が正しく準備できたことを確認はできたのですが、
    操作に慣れるためにもう少し触ってみましょう。
    先程の表示の下に、さらに文字入力ができるようになっていると思います。このように、
    端末では、文字を入力し、エンターキーを押す → 何かしらの文字が表示される → 文字を入力し、
    エンターキーを押す → 何かしらの文字が表示される →……を繰り返して操作していきます。
    では、今度は以下のように入力してみてください。「プロンプトは入力しない」のは覚えていますか？
    入力が終わったら、最後にエンターキーを押してください。

    ```
    bash-3.2$ python3 -h
    ```

    なにやらたくさん表示されましたね！これは「ヘルプ」と呼ばれるもので、
    pythonの後にどんなものを入力することができるかが書かれています（-hは help の h）。
    今は意味はわからなくて全然構いません。

    Step4 コマンドラインでのファイル操作
    これから様々なアプリケーションを作成していくと思いますが。
    どれも 1 つのフォルダにまとめてデータが格納されています。
    その格納する時にはコマンドラインと言うものを使います。
    例えば家を作る時は正しい住所に移動してから家を作りますよね？
    間違って急斜面や周りに何もない畑に作ってしまったら大変なことになりますね。
    コマンドラインでの操作も一緒です。
    自分が作りたい正確な場所に移動をして、フォルダを作成しその中にフォルダの部品となる
    ファイルを作成していきます。
    ……とその前に、基礎的な単語を勉強しておきましょう。
    後ほど実際に端末を触りながらディレクトリやファイルを操作しますが、
    その前に前提知識となるいくつかの概念を説明します。

    フォルダ/ディレクトリ
    最初に出てきたフォルダについてです。フォルダとはディレクトリの別名にもなり簡単に言うとフォルダとは入れ物です。
    何の入れ物かというと後に出てくるファイルと言うものの入れ物に当たります。
    しかし実体はありませんが、その代わりフォルダの中にフォルダを作成することが出来ます。

    ファイル
    その中のファイルとはデータやプログラムの基本単位です。
    そしてこれは拡張子と呼ばれる文字列が割り当てられ、それに応じたプログラムが入っています。
    これらのファイルがフォルダに集まり 1 つのアプリケーションとなります。

    パス
    次にパスについてです。パスとは先ほどの例に出てきた住所みたいなものです。
    フォルダやファイルの保存場所までの道筋を表しています。
    そしてパスは絶対パスと相対パスという 2 種類のパスがあります。
    絶対パスと言うのは、ルートディレクトリと呼ばれるディレクトリの頂点となる所からの
    アクセスするためのものです。
    例えるなら誰が見ても変わらない住所の事です。「東京都豊島区南池袋一丁目」と
    書けば東京都の人も北海道の人が見てもわかりますよね。
    このように絶対パスとは、どこからどんな人が見てもわかるパスの事です。
    次に相対パスは自分がいるところからの目的のフォルダやファイルの場所までの経路を表すものです。
    相対パスを例えるなら自分から見た住所です。例えば目的地が「東京都豊島区南池袋一丁目」だとします。
    仮に自分が豊島区に住んでいたら「南池袋一丁目」になります。
    もしくは北海道に住んでいるのであれば「東京都豊島区南池袋一丁目」となります。
    絶対パスは誰が見てもどこにあるかわかるのですが、相対パスに比べて文字数が増えわかりにくくなります。
    相対パスは絶対パスに比べてパスの文字数が少なく済みます。その変わり相対パスは自分がいる
    場所が変わってしまったらその都度変更する必要があります。
    どちらもメリットデメリットがあるので状況に応じてどちらも使えるようになりましょう。

    と書くことで複数のフォルダを作れます。

    ファイルを移動させる
    まずstyle.cssをcodevillageディレクトリで作ってみてください!
    そしたら今作ったstyle.cssを先ほど作った css フォルダに移動させてみましょう。
    $ mv style.css ./css
    mv ［移動させたいファイル名］［移動先ディレクトリ］
    移動したか確認するため css フォルダに移動してファイルを確認してみて下さい。
    そこでstyle.cssができていたら完了です!
    作成したcssディレクトリにカレントワーキングディレクトリを変更する
    つぎのリネームに移る前に、またカレントワーキングディレクトリを先程作ったディレクトリに変更しておきましょう。
    おさらいです。カレントワーキングディレクトリを 先程作成したcssディレクトリに変更するコマンドは？
    解答

    カレントワーキングディレクトリを変更してから次に進みましょう。
    リネーム
    では今移動させたstyle.cssをindex.cssに変更してみましょう
    名前を変更する時にもmvコマンドを使う。

    ```
    $ mv style.css index.css
    ```

    mv元ファイル名 新しいファイル名
    これでindex.cssに変更されていたら成功です。
    削除
    次にファイルを削除してみましょう。
    以下を実行してみて下さい。

    ```
    $ rm index.css
    ```

    これで中身を確認してなくなっていたら削除完了です！
    ちなみにディレクトリを削除するコマンドは

    ```
    $ rm -r ディレクトリ名
    ```

    となります。
    なぜコマンドラインを使うのか？
    1. プログラムは基本的にファイルの操作だということ→ コマンドラインは
    2. ファイル操作においては最高のツールです。プログラミングの基本はファイル操作です。
    これまで学習してきた mkdir コマンドや
    3.  rm コマンドなど、ほとんどがファイル操作なのです。
    そしてこれから書くプログラムもファイルに書きます。その後ファイルに書いたものを
    コンパイルと呼ばれる自分達が書いたプログラムをコンピューターが読めるように変換します。
    これもファイル操作の一種なのです。
    これだけたくさんファイル操作をするのでより細かい動きができるコマンドラインが好まれるのです。
    2. 人とコンピュータとのやりとりをしやすくするため→ プログラムのなかでは、共通言語です。
    そしてコマンドラインで操作することはコンピューターと共通言語でコミュニケーションをとっている事になるのです。
    もし mac と windows の場合、GUI だと同じ操作を共有するのが難しい場面がしばしばあります。
    その際 CUI であるコマンドラインで操作すればそれらを防ぎ、同じ文字列を打てば同じ作業を行う事が出来ます。
    これらの理由から開発者はコマンドラインを頻繁に使います。
    慣れるまでは使いづらさを感じるかもしれませんが、早いうちに慣れておきましょう。

    ### Step5 エディタのセットアップ
    スクールでは、Visual Studio Code（以下 VS Code）というエディタを使います。
    VS Code はみなさんの PC には普通は入っていないので、別途インストールする必要があります。
    以下のページからインストーラをダウンロードして、各自インストールしてください。
    Download Visual Studio Code
    注意点:
    * Windows の場合: 途中の「追加タスクの選択」で必ず「PATH への追加」にチェックを入れてください。
    * macOS の場合:
        1. VS Code を起動し、Shift+⌘+Pを押す。
        2. 表示された入力欄にshell commandと入力する。
        3. 候補に表示されたShell Command: Install 'code' command in PATHを選択し、エンターを押す。
        4. ターミナルを再起動して、code .で VS Code が起動することを確認する。

    VS Code の立ち上げ方
    それでは、VS Code を立ち上げてみましょう。まず、端末で、
    カレントワーキングディレクトリを先程作ったcodevillageディレクトリに変更しておいてください
    （先程からそのままになっていれば大丈夫です）。カレントワーキングディレクトリの確認方法は覚えていますか？
    次に、以下のコマンドを実行して、VS Code を立ち上げます。
    
    ```
    $ code .
    ```

    .が見えにくいですがこれも必ず入力してください。codeと.の間はスペースを空けることを忘れずに。
    今後、VS Code を立ち上げるときは毎回このコマンドを使いますので、
    「VS Code を立ち上げてください」と言われたときは、端末でcode .を実行する、と覚えておいてください。
    さて、なにやらイケイケなウィンドウが表示されましたね。
    これが VS Code というテキストエディタです。では早速、テキストファイルの作成方法を学んでいきましょう。

    ### Step6 VS Code でテキストファイルを作る
    VS Code は GUI プログラム（前回やりましたね）なので、これまで使っていた端末とは違い、
    マウスで操作することも可能です。文明開化です。
    では、テキストファイルを作ってみましょう。まずはお試しということで、
    適当な文字を書いたテキストファイルを作ることにします。
    VS Code を起動した状態で以下のような表示になっていると思います
    （色は違いますが気にしないでください; あとで自分好みにカスタマイズすることもできます）が、
    下記の赤い円で囲んだアイコンをクリックしてください。これで、新しいファイルを作成する準備ができます。

    -画像-

    アイコンをクリックすると、入力欄が表示されます。これはこれから作るファイルの名前を入力することを求められています。
    今回はsample.txtと入力してください。入力が終わったらエンターキーを押してください。
    すると、右側の領域に「sample.txt」と書かれたタブが作られて、そこに何か入力できるようになったと思います。
    ここに何か書いて保存すれば、sample.txtというテキストファイルにその内容が保存されることになります。
    なんでもいいので、適当に文字を入力してみてください。
    入力が終わったら、macOS なら⌘S、Windows ならCtrl+Sを押して、保存してください。
    「ファイル」メニューからも保存はできますが、テキストエディタで「保存」は
    何千何万回と使う操作ですので、なるべく早い段階でキーボードショートカットを覚えておいたほうが良いでしょう。
    これでテキストファイルが作成できました！なんともシンプルではなかったでしょうか。

    このようにひたすら文字を編集することしかできなくなっています。これがテキストエディタです。

   ### Step7 VSCode 便利プラグインを導入しよう
    日本語翻訳
    VSCode の言語がデフォルトで英語に設定されています。
    日本語設定に変更するためにはプラグインと呼ばれる拡張機能を導入する必要があります。
    インストールの方法は簡単です。VSCode のサイドバーの四角形が
    書けたアイコンをクリックすると検索バーが出てきます。

    その検索バーでjapanese languageと検索して一番上に出てきた
    このプラグインのインストールボタンを押してダウンロードしてください。
    (既にインストールしてる場合はアンインストールと表示されます。)

    インストール後は、Visual Studio Code の再起動が必要です。
    インストール後、画面右下にポップアップが表示されますので、ポップアップ内の
    「Restart Now」をクリックすれば完了です。
    ※もしくは一旦 Visual Studio Code を閉じて再度立ち上げて下さい。
    再起動後、日本語表記になっていることを確認してください。
    Live Server（自動でブラウザ更新）
    次にオススメのプラグイン はLive Severです。
    LiveServer をインストールすると VSCode でコードを保存するとすぐにブラウザに反映することが出来ます。
    Web サイトの作成だけではなく、アプリケーション開発の際にも役に立つので必ず導入しておきましょう。
    先ほどと同じようにサイドバーの四角形のようなボタンをクリックすると左側に検索バーが出てきます。
    検索バーでLive Serverと検索して一番上に出てきたプラグインのインストールボタンを押してダウンロードしてください。
    (既にインストールされてる場合はアンインストールと表示されます。)

    次に Live Server の使い方を説明していきます。まずは、
    VSCode から HTML ファイルが格納されているフォルダを開きます。(HTML ファイルだけだと使用できない場合があります)
    フォルダを開くと、画面右下に「Go Live」という表記が現れます。これをクリックします。
    もしなかった場合はAlt+Lの 2 つを押した後Alt+Oで立ち上げる事が可能です。

    するとローカルサーバーが立ち上がり、ブラウザで HTML が反映された Web ページが開きます。
    コードを修正していく中で HTML や CSS ファイルを保存すると自動でブラウザが更新されるようになります。

    Live Server を使っていない場合は、ファイルを修正して保存する度に、
    ページを更新する必要がありました。ですが Live Server を使用すれば修正した内容が自動で 
    WEB ページに反映されるので、毎回手動で確認する手間が省けます。
    少ない作業でコードの確認ができるので、とても便利なツールです。なお、ローカルサーバーを停止したい時は、
    VSCode の右下にある「Port:5500」と書かれた所を押すとローカルサーバーを終了することが出来ます。
    Ubuntu の日本語設定(Windows の方向け)
    後から追加した内容のため動画に反映されていません。ご不便おかけして申し訳ございませんが、
    何かわからないことがあれば、個別指導にお越しください。
    ここでは Ubuntu の使用言語を日本語にしていきます。
    ホームディレクトリに戻って以下のコマンドを実行して下さい。ctrl + c でコピー ctrl + v でペーストすることができます

    ```
    $ sudo apt update
    ```

    次に Ubuntu 用の日本語パックをインストールする以下のコマンドを実行します。

    ```
    sudo apt -y install language-pack-ja
    ```

    そして日本語パックを使って Ubuntu の表記を日本語にします。

    ```
    sudo update-locale LANG=ja_JP.UTF8
    ```

    これが終わったら一度 Ubuntu を閉じて再起動してください。(もし変わらなかったら PC の再起動もしてみて下さい。)
    Ubuntu を再起動して日本語になっていたら完了です。折角なので日本語のマニュアルもインストールしておきましょう。

    ```
    sudo apt -y install manpages-ja manpages-ja-dev
    ```

    以上で日本語設定が完了しました。
